# 함께 여행하고, 기록하고, 나누며, 성장할 수 있도록 돕는 여행 동행 플랫폼

## 1. 🎯기획 의도

![슬라이드2](https://github.com/user-attachments/assets/1789b3a8-cd31-4dd3-9b78-4858ccf64d91)

청년층은 실제 직무를 경험하고 싶어하지만,
현실에서는 다음과 같은 어려움에 부딪히고 있습니다.

-   🧱 경력직 중심 채용으로 인한 진입 장벽
-   📉 경험 부족으로 인한 경쟁력 약화
-   🕳️ 경력을 쌓을 기회 자체의 부재

반면, 청년들은 단순한 취업 준비보다 “현장을 직접 체험하고, 가능성을 증명하며, 성장의 기회를 만드는 과정”을 원합니다.

이에 따라 **KOK(콕)**은

> “경험의 공백을 메우는, 청년을 위한 직무 체험 플랫폼”으로 기획되었습니다.

## 2.💡 기대 효과

![슬라이드4](https://github.com/user-attachments/assets/8d37c8b4-a4dc-4947-8ffb-e31016615c08)


### 1️. 실무 기반의 역량 강화

-   실제 기업의 공백 기간을 활용해 직무를 직접 체험
-   이력서보다 강력한 현장 경험 확보

### 2️. 채용과 연결되는 경험

-   체험을 통해 기업과 자연스럽게 연결
-   단순 서류 평가가 아닌 실제 업무 기반의 채용 기회 제공

### 3️. 커뮤니티를 통한 정보 확장

-   참여자 간 경험 공유 및 조언 교류
-   실질적인 정보와 네트워크 형성

### 4️. 청년 고용 환경 개선

-   경험의 장벽을 낮추고, 청년층의 취업 가능성과 기회를 확대

## 7. 📝총평

### 🌟파견 팀 협업

이번 프로젝트에서 저는 백엔드 개발자로서 타 팀에 파견되어 협업하는 특별한 경험을 하게 되었습니다. 
초기에는 해당 팀의 기획 방향과 데이터베이스 구조에 대한 정보가 부족해 전체적인 흐름을 파악하는 데 어려움이 있었지만, 파견 팀의 기존 코드를 분석하고 기능별 로직을 하나씩 뜯어보며 이해를 넓혀갔습니다.
특히 코드리뷰 과정에서 많은 인사이트를 얻을 수 있었습니다.  

파견 팀의 코드 스타일, 설계 방식, 예외 처리 흐름 등을 면밀히 살펴보며 단순히 기능 구현을 넘어서 왜 이런 구조로 작성되었는지, 어떤 의도를 담고 있는지를 파악하려 노력했습니다.  
리뷰 중에는 중복 로직이나 개선 가능한 부분을 발견해 의견을 제시했고, 실제로 일부 로직을 리팩토링하거나 성능을 고려한 쿼리 수정 제안을 반영하기도 했습니다.
이러한 리뷰 과정은 단순한 코드 분석을 넘어 팀의 개발 문화와 기술적 기준을 이해하고 존중하는 태도를 기를 수 있는 계기가 되었으며, 동시에 제 의견을 논리적으로 전달하고 피드백을 주고받는 커뮤니케이션 능력도 함께 향상되었습니다.  
또한 서비스의 세부 요구사항은 팀원들과의 지속적인 대화를 통해 자연스럽게 파악할 수 있었고, 이를 바탕으로 기능 구현뿐 아니라 기획 의도에 부합하는 백엔드 로직을 설계하고 개선하는 데 기여할 수 있었습니다.  
이번 협업을 통해 느낀 가장 큰 교훈은 코드리뷰와 커뮤니케이션이 협업의 핵심이라는 점입니다.  
실무에 가까운 환경에서 다양한 역할을 경험하며 백엔드 개발자로서의 역량을 점검하고 성장할 수 있었던 값진 시간이었고, 앞으로의 개발 커리어에 있어 중요한 자산이 될 것이라 확신합니다.
